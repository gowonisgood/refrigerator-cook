프로젝트 목표 (Goal)
사용자가 보유한 재료를 입력하면 LangGraph ReAct 패턴을 활용하여 요리 레시피를 추천해주는 AI Agent를 Streamlit으로 구현한다.

TheMealDB API로 먼저 레시피를 검색하고, 결과가 없을 경우 LLM이 직접 레시피를 생성하는 분기 워크플로우를 포함한다.

2
구현할 Tool 리스트
🔍
search_mealdb
Primary Tool — 메인 레시피 검색
TheMealDB API를 통해 재료명(영어)으로 레시피를 검색합니다. 상위 2개 레시피의 상세 조리법과 재료 목록을 반환합니다. 결과가 없을 경우 MEALDB_NOT_FOUND 태그를 반환하여 Agent가 폴백 경로로 분기합니다.
API: TheMealDB v1 입력: ingredient (영어) 인증: 불필요 무료
📊
get_nutrition_info
Secondary Tool — 영양 정보 조회
Open Food Facts API로 재료의 영양 성분을 조회합니다. 100g 기준 칼로리·단백질·탄수화물·지방을 반환합니다. 정보를 찾지 못하면 LLM 추정값 사용을 안내합니다.
API: Open Food Facts 입력: ingredient (한/영) 인증: 불필요 무료
🌐
search_web_recipe
Auxiliary Tool — 웹 실시간 검색
Tavily Search API로 실시간 웹에서 레시피를 검색합니다. TheMealDB 실패 시 보조 수단으로 활용되며, 최신 트렌드 레시피나 특수 요리 정보를 보완합니다.
API: Tavily Search 입력: query (키워드) 인증: TAVILY_API_KEY 무료 월 1,000회
✨
generate_recipe_with_llm
Fallback Tool — LLM 직접 생성
TheMealDB 검색 실패 시 LLM(gpt-4o-mini)이 직접 레시피를 생성하는 폴백 도구입니다. 입력 재료와 제외 재료를 고려한 한국어 레시피를 구조화된 형식으로 반환합니다.
API: OpenAI gpt-4o-mini 입력: ingredients, excluded 인증: OPENAI_API_KEY 유료 (종량제)
3
LangGraph 워크플로우 시각화
시작/종료 Agent 노드 (LLM) Tool 노드 조건 분기 출력 노드
✅ TheMealDB 성공 경로
search_mealdb → 레시피 발견
→ get_nutrition_info 영양 조회
→ Agent가 한국어로 정리하여 출력
⚠️ TheMealDB 실패 → 폴백 경로
search_mealdb → MEALDB_NOT_FOUND
→ generate_recipe_with_llm 직접 생성
→ get_nutrition_info → 최종 출력

수용 기준 (Acceptance Criteria)
  • 1
    재료를 텍스트 자유 입력 또는 체크박스/드롭다운 선택으로 입력할 수 있다
  • 2
    알레르기/거부 재료를 선택하면 추천 레시피에서 해당 재료가 제외된다
  • 3
    Agent가 TheMealDB API로 레시피를 먼저 검색한다
  • 4
    TheMealDB에서 결과가 없을 경우 LLM이 직접 레시피를 생성하는 분기가 동작한다
  • 5
    최종 출력에 레시피명 · 재료 목록 · 조리 순서 · 영양 정보 · 조리 시간이 모두 포함된다
  • 6
    Streamlit UI에서 Agent 실행 단계(Thought / Action / Observation)가 실시간으로 표시된다
  • 7
    영양 정보는 Open Food Facts API 또는 LLM 추정값으로 제공된다
  • 8
    Streamlit Cloud에 requirements.txt 기반으로 배포 가능하다
5
기타 요구사항

🛠️ 기술 제약 (Constraints)

  • Python ≥ 3.10
  • LangGraph + LangChain 기반 ReAct Agent
  • LLM: gpt-4o-mini (최저가)
  • 무료 API 우선, 유료 API 최소화
  • Streamlit Cloud 배포 가능 구조
  • 모든 UI · 출력 한국어

🔑 환경 변수 (Secrets)

  • OPENAI_API_KEY — 필수
  • TAVILY_API_KEY — 선택 (없어도 동작)
  • 로컬: .env 파일 사용
  • Cloud: Streamlit Secrets 대시보드

🗂️ 프로젝트 구조

  • app.py — Streamlit 진입점
  • agent/tools.py — 4개 도구
  • agent/prompts.py — 시스템 프롬프트
  • agent/core.py — Agent 팩토리
  • requirements.txt — 의존성
  • .streamlit/config.toml — 테마

📦 주요 의존 패키지

  • langgraph ≥ 0.2
  • langchain ≥ 0.3
  • langchain-openai ≥ 0.2
  • tavily-python ≥ 0.5
  • streamlit ≥ 1.40
  • requests · python-dotenv

도메인 데이터 모델 (Ontology Schema)
📥 UserInput — 사용자 입력
ingredients List[str] 보유 재료 목록 예: ["당근", "계란", "밀가루"]
excluded_ingredients List[str] 알레르기 또는 거부 재료 목록
🍽️ RecipeResult — 최종 추천 레시피
recipe_name str 추천 요리 이름
recipe_ingredients List[str] 요리에 필요한 재료 목록 (분량 포함)
cooking_steps List[str] 단계별 조리 순서
nutrition NutritionInfo 영양 정보 (칼로리·단백질·탄수화물·지방)
cooking_time CookingTime 준비 시간 + 조리 시간
source str "TheMealDB" | "LLM_Generated" — 레시피 출처 태그
📊 NutritionInfo — 영양 성분
caloriesfloat칼로리 (kcal)
protein_gfloat단백질 (g)
carbs_gfloat탄수화물 (g)
fat_gfloat지방 (g)
⏱️ CookingTime — 조리 시간
prep_minutesint재료 준비 시간 (분)
cook_minutesint실제 조리 시간 (분)
